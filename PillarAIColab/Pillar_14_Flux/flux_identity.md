# Pillar 14: Flux (The Rate of Transfer)

## 📂 1. Philosophy: Bandwidth and Movement
* **Core Logic**: Flux is the volume and speed of information or energy passing through the identity’s boundaries.
* **Functional Role**: It measures throughput and determines whether the system can move data fast enough to sustain complex delegations.
* **Active State**: Flux represents the current/active state of movement; without it, the agent is effectively frozen.

## 📂 2. Economic: The Coefficient of Transfer
* In the Net Value Equation, Flux is a multiplier for effective production and task execution throughput.
* **Bandwidth**: High Flux enables larger context windows and more complex planning without bottlenecking.
* **Throughput Gate**: Low Flux forces the system to throttle output, even when other pillars like Force or Willpower are strong.

## 📂 3. Logic: Functional Mechanics
* **Transfer Rate**: Throughput scales with Flux, allowing more information to flow per timestep.
* **Flow Stability**: When Flux exceeds Depth, the system enters a turbulent state and may lose coherence.
* **Recollection Speed**: Flux interacts with Memory to determine how quickly historical context is pulled into working memory.

## 📂 4. Interaction: How Flux Touches Others
| Interaction | Result |
| :--- | :--- |
| **Flux + Force** | **Momentum**: High Flux allows Force to be applied across more timesteps, increasing effective execution speed. |
| **Flux + Memory** | **Recollection Speed**: High Flux enables faster retrieval of history into the current context window. |
| **Flux + Depth** | **Stability Check**: If Flux outpaces Depth, the agent risks turbulence and overload. |

---

## 🛠️ Folder-Level "Blackboard" Entry
**Primary Directive**: Keep throughput high enough to support task complexity, but balanced with Depth.
* **Management Fix**: When bottlenecks occur, raise Flux or offload work to prevent the PCIe bus from becoming a throughput choke point.
* **Warning**: High Flux with low Depth causes turbulence and unstable behavior.
